#### ✅ 编写 user 云函数 — 用户资料
- **时间**：2026-06-12 12:51
- **内容**：`getProfile` 获取用户信息+小组信息+成员列表，`updateProfile` 更新昵称/头像
- **涉及文件**：`cloudfunctions/user/index.js`、`package.json`、`config.json`
- **验证要点**：getProfile 返回完整的用户+小组数据，updateProfile 成功更新

#### ✅ 编写 group 云函数 — 小组管理
- **时间**：2026-06-12 12:51
- **内容**：
  - `create`：校验用户无小组 → 生成唯一6位邀请码（30字符集排除0/O/1/I/L）→ 创建小组 + 用户加入
  - `join`：校验用户无小组、小组未满10人 → 加入
  - `leave`：最后一人退出时自动解散小组 + 逻辑删除共享待办
  - `detail`：获取小组信息+成员列表
  - `resetCode`：仅创建者可重置邀请码
- **涉及文件**：`cloudfunctions/group/index.js`、`package.json`、`config.json`
- **验证要点**：创建小组→生成邀请码→另一个用户通过邀请码加入→退出→解散流程完整

#### ✅ 编写 task 云函数 — 待办管理
- **时间**：2026-06-12 12:51
- **内容**：
  - `create`：校验 GROUP 需有小组 → 创建待办
  - `update`：编辑标题/描述/优先级/截止时间等
  - `delete`：逻辑删除（isDeleted=true）
  - `toggle`：TODO↔DONE 切换，完成时自动通知组员
  - `switchVisibility`：PERSONAL↔GROUP 切换
  - `list`：筛选（状态/优先级/可见性）+排序（创建时间/截止时间/优先级）+分页
  - `detail`：单条查询+访问控制（个人任务仅自己，共享任务组员可见）
  - `getDates`：获取月份中有待办的日期列表
  - `getByDate`：获取指定日期的待办列表
- **涉及文件**：`cloudfunctions/task/index.js`、`package.json`、`config.json`
- **验证要点**：CRUD+状态切换+可见性切换全流程，筛选排序分页正确
