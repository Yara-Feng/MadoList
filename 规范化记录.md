## 三、技术架构

### 3.1 项目结构

```
MadoList/
├── cloudfunctions/          # 云函数（action-based dispatch）
│   ├── login/              # 微信登录 + 自动注册
│   ├── user/               # 用户资料 CRUD
│   ├── group/              # 小组管理（创建/加入/退出/详情/重置邀请码）
│   ├── task/               # 待办管理（CRUD/状态切换/可见性切换/列表/日期查询）
│   ├── reminder/           # 提醒管理（创建提醒/防骚扰检查）
│   └── notification/       # 通知管理（列表/标记已读/未读计数）
├── miniprogram/
│   ├── app.js              # 全局状态管理（userInfo/groupInfo/unreadCount）
│   ├── app.json            # 路由配置 + TabBar（4个Tab）
│   ├── app.wxss            # 全局样式（CSS变量主题系统）
│   ├── utils/
│   │   ├── api.js          # 云函数调用封装（userApi/groupApi/taskApi/reminderApi/notificationApi）
│   │   ├── constants.js    # 常量定义（枚举值/配置）
│   │   └── date.js         # 日期工具函数
│   ├── components/
│   │   ├── task-card/      # 待办卡片组件
│   │   └── empty-state/    # 空状态组件
│   └── pages/
│       ├── home/home       # 首页（用户信息/小组状态/快捷操作/今日待办）
│       ├── tasks/tasks     # 待办列表（筛选+排序+分页）
│       ├── tasks/detail    # 待办详情（查看/编辑/创建/完成/删除/提醒）
│       ├── calendar/       # 日历视图（月视图+日期标记+显示模式切换）
│       ├── group/group     # 小组主页
│       ├── group/create    # 创建小组
│       ├── group/join      # 加入小组
│       ├── group/members   # 成员列表
│       ├── notifications/  # 通知中心
│       └── profile/profile # 我的（个人资料/小组信息）
```

### 3.2 数据库设计

| 集合 | 说明 | 关键字段 |
|------|------|----------|
| users | 用户信息 | _openid(唯一), nickname, avatarUrl, groupId |
| groups | 小组信息 | inviteCode(唯一), name, createdBy |
| tasks | 待办事项 | _openid, groupId, visibility, priority, status, dueTime, isDeleted |
| reminders | 提醒记录 | taskId, senderOpenId, receiverOpenId, createdAt |
| notifications | 通知中心 | receiverOpenId, type, title, content, isRead, relatedTaskId |
| checkins | 打卡记录（预留） | taskId, _openid, date |

### 3.3 索引设计

| 集合 | 索引 |
|------|------|
| users | _openid(唯一), groupId |
| groups | inviteCode(唯一) |
| tasks | (_openid, status, isDeleted), (groupId, visibility, status), dueTime, createdAt |
| reminders | (taskId, senderOpenId, createdAt) |
| notifications | (receiverOpenId, isRead, createdAt) |

## 四、开发规范

### 4.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件/目录 | kebab-case | `task-card/`, `empty-state/` |
| 云函数模块 | 小写单词 | `login`, `user`, `group` |
| 数据库集合 | 小写复数 | `users`, `groups`, `tasks` |
| 字段名 | camelCase | `avatarUrl`, `groupId`, `createdAt` |
| 常量 | UPPER_SNAKE_CASE | `GROUP_MAX_SIZE`, `ANTI_SPAM_INTERVAL` |
| CSS 类名 | kebab-case | `task-card`, `tag-priority-high` |

### 4.2 代码风格

- **云函数**：action-based dispatch 模式，每个 action 对应一个 async 函数
- **前端**：Page 模式，数据驱动视图，事件委托处理交互
- **错误处理**：云函数统一返回 `{ success: boolean, data?: any, errMsg?: string }`
- **日志**：云函数使用 `console.error` 记录错误，前端使用 `console.error` 并显示 Toast

### 4.3 API 调用规范

所有云函数调用统一通过 `utils/api.js` 封装，格式为：
```javascript
const { moduleApi } = require('../../utils/api');
const result = await moduleApi.method(params);
```
每个 API 模块按业务领域划分：userApi / groupApi / taskApi / reminderApi / notificationApi。
